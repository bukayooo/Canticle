# Canticle

An iOS devotional app built on the 1662 Book of Common Prayer. On launch, Canticle checks the
clock and the calendar and shows you today's Morning or Evening Prayer — the fixed liturgy plus
the day's appointed Psalms and Lessons — in a single scrollview.

## Project layout

Open `Canticle/Canticle.xcodeproj` in Xcode. The project file is generated by
[XcodeGen](https://github.com/yonaskolb/XcodeGen) from `Canticle/project.yml` — if you change
targets, sources, or build settings, edit `project.yml` and run `xcodegen generate` from the
`Canticle/` directory rather than hand-editing the `.xcodeproj`.

**This also applies to `Resources/` contents**, not just Swift source files: `project.yml`'s
`resources:` entries are enumerated into individual "Copy Bundle Resources" file references at
generation time, not watched live. Adding, removing, or renaming *any* file under `Canticle/Canticle/
Resources` (e.g. regenerating the Bible/Psalter/Calendar JSON) leaves the `.xcodeproj` stale until
you run `xcodegen generate` again — a stale reference to a deleted file fails the build with
"No such file or directory"; a new file silently isn't compiled/copied at all until then.

```
Canticle/
  project.yml                  — XcodeGen project spec
  Canticle/
    App/CanticleApp.swift      — @main entry point
    Models/                    — Office, LiturgyItem, DailyReadings, PsalmText, BibleBook, ScriptureRef
    Services/
      TimeOfDayService.swift   — decides Morning vs Evening from the clock; computes day-of-year/month
      CalendarStore.swift      — loads Resources/Calendar/daily_readings.json (Lessons + Collect)
      PsalterStore.swift       — loads Resources/Psalter (all 150 psalms + the 30-day cycle)
      BibleStore.swift         — loads Resources/Bible on demand; resolves Lesson references to text
    Liturgy/
      MorningPrayer.swift       — fixed 1662 Order for Daily Morning Prayer
      EveningPrayer.swift       — fixed 1662 Order for Daily Evening Prayer
      ApostlesCreed.swift       — shared text, said at both offices except appointed feasts
      AthanasianCreed.swift     — said at Morning Prayer instead, on its appointed feasts
      Litany.swift              — said after Morning Prayer on Sundays/Wednesdays/Fridays
      Commination.swift         — occasional penitential service (Ash Wednesday)
    ViewModels/DevotionalViewModel.swift — merges the fixed liturgy with today's Psalms/Lessons/Collect
    Views/                     — ContentView (the one scrollview) + components
      LitanyView.swift, ComminationView.swift — standalone screens for the occasional services
    Theme/Theme.swift          — Crusader-inspired palette & typography
    Resources/
      Psalter/                 — the full 1662 Psalter (see below) — ships complete, not a sample
      Bible/                   — KJV + Apocrypha text, one JSON file per book — ships complete
      Calendar/daily_readings.json — the full 365-day fixed Kalendar of Lessons (see below)
    Canticle.icon               — app icon (Icon Composer): gold/black gradient, red Templar cross
```

## How "today's office" is decided

`TimeOfDayService.currentContext(now:)` reads the system clock:
- Before 12:00 noon → **Morning Prayer**. From noon onward → **Evening Prayer**.
- `dayOfYear` (1–366) is computed via `Calendar.ordinality(of: .day, in: .year, for:)` — this is
  the key the Lessons/Collect calendar JSON is looked up by.
- `dayOfMonth` (1–31) is used separately to look up the Psalter cycle (see below) — the Psalter
  doesn't depend on the calendar date at all.

`DevotionalViewModel` takes the fixed liturgy for whichever office is active and splices in the
day's Psalms, First and Second Lessons, and Collect of the Day at the appropriate points.

In DEBUG builds, a segmented control at the bottom of the screen lets you preview the other office
without waiting for the clock (`ContentView.debugOfficeSwitcher`, wrapped in `#if DEBUG`).

## The Psalter (complete, not a sample)

Unlike the Lessons calendar, **the Psalter ships complete** — all 150 psalms and the full 30-day
reading cycle, in `Resources/Psalter/psalms.json` and `psalter_cycle.json`. This is possible
because, unlike the lectionary (which is keyed to specific calendar dates and moveable feasts), the
1662 Psalter is read on a fixed monthly cycle that repeats every month regardless of the date —
`PsalterStore.psalms(dayOfMonth:office:)` looks up the appointed psalms for whichever day-of-month
it is. Per the Psalter's own rubric, the 31st day reuses the 30th day's psalms (there's no separate
entry for it).

The text and cycle were sourced from the 1662 Psalter transcription at
[eskimo.com/~lhowell/bcp1662/psalter](https://www.eskimo.com/~lhowell/bcp1662/psalter/) (5 pages,
one per week) and cross-checked against a scan of an original 1662 Kalendar/Psalter — the day/psalm
groupings and wording match exactly. Psalm 119 is long enough that it's read across five
consecutive offices (Day 24 Evening through Day 26 Evening) rather than repeated in full each
time — `PsalterStore` and the parsing that built the JSON both account for this.

## The Bible text and Lesson references

`ScriptureReading` (in the calendar JSON) holds only a bare reference like `"Genesis 1:1-19"` —
no embedded verse text. `DevotionalViewModel` parses it with `ScriptureRef` and resolves the actual
text at render time via `BibleStore`, which loads the relevant book's JSON file from
`Resources/Bible` on demand (each book is its own file, so opening the app doesn't load the whole
Bible into memory at once).

- All 80 books (the 66 canonical books plus the 14 Apocrypha/Deuterocanon books some autumn
  Lessons are appointed from — Ecclesiasticus, Wisdom of Solomon, Tobit, Judith, Baruch, Susanna,
  Bel and the Dragon, 1/2 Maccabees, 1/2 Esdras, Additions to Esther, Prayer of Azariah, Prayer of
  Manasseh) come from the same source: the King James Version + Apocrypha, 1769 standardized text,
  in USFX format from [eBible.org](https://ebible.org) (public domain; courtesy of the Crosswire
  Bible Society). This replaced an earlier two-source setup (modern-spelling KJV for the 66 canon,
  archaic 1611-spelling Apocrypha from a different source) that had a visible spelling
  inconsistency between the two — everything is modern spelling now, from one consistent source.
  `parse_usfx.py` (scratchpad-only, not shipped) walks the USFX XML's verse/chapter milestones,
  strips footnotes and the KJV's "¶" paragraph-break marks, and writes the same per-book JSON
  shape as before.
- `BibleStore.canonicalBookName(for:)` resolves common abbreviations (the forms used in the 1662
  Kalendar, e.g. "Gen.", "1 Cor.", "Ecclus.") to the canonical book name — see the `bookAliases`
  table in `BibleStore.swift`. Extend that table if a reference in your calendar data doesn't
  resolve.
- If a reference can't be parsed or the passage isn't found, `DevotionalViewModel` falls back to
  the same "not yet added" empty state used for missing calendar data (`MissingDailyContentView`)
  rather than showing nothing or fabricated text.

## The Kalendar of Lessons (complete, with known limitations)

`Resources/Calendar/daily_readings.json` now ships with **all 365 days** of the fixed 1662
Kalendar's Table of Lessons, sourced from the Table of Lessons at
[eskimo.com/~lhowell/bcp1662/info/cal_1871](https://www.eskimo.com/~lhowell/bcp1662/info/cal_1871/)
(one page per month) and parsed straight from each lesson link's URL, which conveniently encodes
the full canonical reference (e.g. `Genesis 17:19-27`) regardless of how abbreviated the visible
link text is — see `scratchpad`-only `parse_kalendar.py` used to build the JSON (not shipped in the
app). Each entry has this shape:

```json
{
  "dayOfYear": 1,
  "dateLabel": "1 January",
  "feastName": "Circumcision of our Lord",
  "isRedLetterDay": true,
  "firstLessonMorning":  { "reference": "Genesis 17:19-27" },
  "secondLessonMorning": { "reference": "Romans 2:17-29" },
  "firstLessonEvening":  { "reference": "Deuteronomy 10:12-22" },
  "secondLessonEvening": { "reference": "Colossians 2:8-18" }
}
```

- `feastName`/`isRedLetterDay` are metadata about fixed Holy Days; `collectOfTheDay` is omitted
  entirely for now (see "Known limitations" below) and decodes as `nil` — `DailyReadings.collectOfTheDay`
  is `Collect?`, so days without one show the same "not yet added" placeholder as any other missing
  content.
- References parse as `"Book"` (a single-chapter book referenced by name alone, e.g. "Jude"),
  `"Book C"`, `"Book C:V"`, `"Book C:V-V"`, or a cross-chapter span `"Book C:V-C:V"` — see
  `ScriptureRef.swift`. A handful of Apocrypha Lessons (Wisdom, Ecclesiasticus, Baruch) don't carry
  the book name on every row in the source (it's only spelled out when it changes), so the parser
  tracks it per lesson column; see `parse_apocrypha_cell` in the scratchpad script.
- **Leap years**: the source's February page includes a 29 February row, but it's dropped when
  building the JSON so `dayOfYear` (1–365) lines up with `Calendar`'s ordinality in an ordinary
  year — the common case. In an actual leap year this means every day from 1 March onward is off
  by one dayOfYear until the following year resets it; a known, deliberately accepted tradeoff
  rather than building full leap-year-aware calendar logic.
- One genuine source typo was caught and rejected rather than guessed at:
  `Ezekiel 27:26-21` (7 September, First Lesson Evening) has an impossible backwards verse range;
  `ScriptureRef` validates that end ≥ start and returns `nil` for anything that doesn't, so this one
  day's First Lesson Evening shows the "not yet added" placeholder instead of garbled text.

### Movable feasts (Easter computus)

`Services/MovableFeasts.swift` computes Easter Day for any year using the standard Western
("Anonymous Gregorian"/Meeus-Jones-Butcher) algorithm — a well-known mathematical procedure, not
sourced from any particular publication — and derives Ash Wednesday, Ascension Day, Whitsunday,
and Trinity Sunday from it by fixed day-offsets. This unlocks:

- **The Athanasian Creed**'s remaining 4 movable-feast appointments (Easter Day, Ascension Day,
  Whitsunday, Trinity Sunday), alongside the 9 fixed-date ones — `AthanasianCreed.isAppointed(on:)`
  now checks both.
- **A Commination**'s contextual surfacing on Ash Wednesday — `Commination.isAppointed(on:)` now
  drives a card in Morning Prayer's scrollview on that day, the same way the Litany's card works
  (see below), rather than only being reachable from the toolbar menu.

### Sunday Proper Lessons and Collects

`Services/ChurchYear.swift` resolves any date to its named Sunday in the church year (e.g.
"Advent1", "Trinity5", "Easter2") — the same identifiers used as HTML anchors in the source
tables — using `MovableFeasts` for the Easter-anchored portion of the year (Septuagesima through
Trinity) and fixed anchors (Advent Sunday near 30 November, Christmas, the Epiphany) for
Advent/Christmas/Epiphany. This was verified against a full year's worth of Sundays in Python
before porting to Swift (52 Sundays/year, zero unresolved).

- **Lessons**: `Resources/Calendar/proper_sundays.json` (from
  [info/proper_sunday.html](https://www.eskimo.com/~lhowell/bcp1662/info/proper_sunday.html)) has
  the Proper First Lesson for every named Sunday, plus a Proper Second Lesson for the handful of
  major Sundays that override it (Septuagesima, Palm Sunday, Easter Day, Easter1, Whitsunday,
  Trinity Sunday). Everywhere else, the Second Lesson still comes from the ordinary fixed Kalendar
  course for that calendar date — `DevotionalViewModel.properLesson(readings:isFirst:)` falls back
  to it automatically. If a date is a Sunday but doesn't resolve to a named Sunday at all (should be
  essentially never, given the verification above), the "not yet added" placeholder shows instead
  of a wrong or fabricated Lesson.
- **Collects**: `Resources/Calendar/collects.json` (from the 4 sub-pages of
  [communion/readings.html](https://www.eskimo.com/~lhowell/bcp1662/communion/readings.html) —
  `xmas.html`, `lent.html`, `easter.html`, `trinity.html`) has the Collect for every named Sunday
  plus several fixed/movable Holy Days (Christmas, Circumcision, Epiphany, Ash Wednesday, Ascension,
  etc. — see `ChurchYear.namedHolyDay(for:)`). Per the 1662 rubric that a Sunday's Collect "shall
  serve all the week following," `ChurchYear.governingSunday(for:)` resolves the Collect for *every*
  day, not just Sundays, by finding the most recent Sunday on or before that date. A Holy Day's own
  Collect takes precedence over the week's ordinary Sunday when both apply.
  - Only the Collect itself was imported, not the Epistle or Gospel that accompany it in the
    source — those belong to the Communion service, which Canticle doesn't implement (only
    Morning and Evening Prayer exist). Building one out would be the natural way to make use of
    them, if wanted.
  - The Collects table only defines distinct Collects through "Trinity25", while Lessons can name
    Sundays up to "Trinity27" in years with an early Easter — `CollectsStore.collect(for:)` falls
    back to "Trinity25" for "Trinity26"/"Trinity27", matching the rubric's general practice of
    continuing the last Sunday's Collect.
  - Doesn't account for an intervening fixed Holy Day's Collect overriding the *previous* Sunday's
    for the rest of that week (only the day the Holy Day itself falls on) — a documented
    simplification.

### Known simplification: the Evening "or" alternative

The Proper Lessons table gives two alternative Evening Lessons for most Sundays (e.g. "1 Sam. 16
*or* 1 Sam. 17"), matching a historical option between an ordinary/shorter and alternative/longer
reading. `parse_proper_sundays.py` (scratchpad-only) always takes the first-listed option; the
second alternative isn't modeled or stored, since the app has no UI concept of "alternate Lessons."

If you'd rather split the calendar into multiple files (e.g. one per month), you'll need a small
change to `CalendarStore.load(from:)` to merge multiple JSON resources; right now it expects one
file named exactly `daily_readings.json`.

## The Litany and A Commination

Two occasional services from the 1662 book are included as standalone screens (reachable from the
book-icon menu in the toolbar), rather than being part of the single daily scrollview, since both
are long, separate rites rather than day-specific inserts:

- **The Litany** (`Litany.swift`, `LitanyView.swift`) is appointed after Morning Prayer on Sundays,
  Wednesdays, and Fridays — a fixed weekday pattern with no dependency on Easter, so
  `Litany.isAppointed(on:)` works correctly today. On those three weekdays, Morning Prayer's
  scrollview shows a card linking straight to it.
- **A Commination** (`Commination.swift`, `ComminationView.swift`) is appointed for Ash Wednesday,
  a movable feast — since Canticle has no computus (see above), it isn't auto-detected or
  surfaced contextually; it's only reachable from the toolbar menu at any time.

## Content notes

- The fixed Morning/Evening Prayer text in `Liturgy/MorningPrayer.swift` and
  `EveningPrayer.swift`, the Athanasian Creed, the Litany, and the Commination were all
  cross-checked against the authoritative transcription at
  [eskimo.com/~lhowell/bcp1662](https://www.eskimo.com/~lhowell/bcp1662/) (fetched directly, not
  from training knowledge) and corrected against it. That pass caught two real gaps in the original
  scaffold: Evening Prayer's Exhortation was missing a sentence present in Morning Prayer's version,
  and both offices were missing three of the five daily Prayers (for the King's Majesty, the Royal
  Family, and the Clergy and People) between the Grace collect and the Prayer of St. Chrysostom —
  both are now fixed.
- The Psalter and the full 80-book KJV + Apocrypha canon are from verified, authoritative digital
  sources (see above) and should be reliable as-is, in consistent modern spelling throughout.
- The five daily Prayers (King's Majesty, Royal Family, Clergy and People, Chrysostom, Grace) are
  always shown in full at both offices for simplicity; the 1662 rubric technically drops the first
  three when the Litany is said immediately after (see "The Litany and A Commination" above) — this
  simplification isn't implemented.

## Design

The palette and cross-pattée motif (`Theme.swift`, `CrusaderCrossShape.swift`) are matched to the
app icon you supplied (`Canticle.icon`): gold-to-black gradient, red Templar cross. Text uses the
system serif design (San Francisco's serif companion, "New York") throughout for a manuscript
feel, adapting between a parchment/cream light mode and a candlelit near-black dark mode.

## App icon (Icon Composer / `.icon` format)

`Canticle.icon` is included in the project as a folder reference so it's visible in Xcode, but
**xcodegen 2.43 predates this Xcode 26 icon format**. `actool` only resolves
`ASSETCATALOG_COMPILER_APPICON_NAME` against icon sets that live *inside* an `.xcassets` catalog —
a standalone `.icon` folder reference outside one doesn't match, and build would fail with
"None of the input catalogs contained a matching ... icon stack named 'Canticle'" if the setting
pointed at it. `project.yml` currently sets `ASSETCATALOG_COMPILER_APPICON_NAME` to an empty
string so the build succeeds with the default blank icon in the meantime.

To wire up the real icon: open the project in Xcode, go to the target's **General** tab, and use
the **App Icon** picker to select `Canticle.icon`. Xcode's own UI handles the new format's
placement/build-setting wiring correctly (this is exactly the kind of thing that's safest done by
hand rather than guessed at in `project.yml`, since the format is brand new). Note that Xcode may
relocate the `.icon` file itself when you do this — `project.yml`'s `sources` entry for it is
`Canticle.icon` (relative to `project.yml`, i.e. alongside `.xcodeproj`), so if you move the file,
update that path to match or `xcodegen generate` will fail with a "missing source directory" error.

## Bundle identifier & signing

`project.yml` sets `PRODUCT_BUNDLE_IDENTIFIER` to `com.canticle.app` as a placeholder — change it
and set your Team in Xcode's Signing & Capabilities tab before running on a device.
